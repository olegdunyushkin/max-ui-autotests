# Тест-дизайн проекта MAX

## Объект тестирования

- публичная главная страница `https://max.ru`;
- страница загрузки `https://download.max.ru`;
- публичная часть формы входа `https://web.max.ru`.

## Цель

Проверить критичные пользовательские пути, публичный контент и навигацию без регистрации,
реальных учётных записей и отправки СМС. Набор построен так, чтобы каждый параметризованный запуск
был отдельным независимым сценарием.

## Наборы запуска

| Набор | Назначение | Сценариев |
|---|---|---:|
| `smoke` | По одному критичному сценарию для каждого адреса MAX | 3 |
| `regression` | Полное автоматизированное покрытие | 54 |
| `responsive` | Отдельная проверка адаптивной вёрстки | 3 |
| полный | Все автоматизированные проверки | 54 |

Smoke и responsive входят в regression: теги описывают назначение сценариев, а не делят их на
непересекающиеся группы.

## Риски и ограничения

- проверки выполняются на внешнем production-окружении, которое меняется независимо от тестов;
- MAX может временно ограничить частые запросы с одного IP-адреса; такая ситуация определяется
  отдельным предусловием и попадает в категорию Allure «Ограничение внешнего окружения»;
- после обнаружения ограничения оставшиеся тесты текущего запуска пропускаются без новых запросов;
- частота открытия новых страниц ограничена параметром `requestInterval`, при этом тесты не
  зависят от результатов и порядка выполнения друг друга;
- внешние магазины могут изменять адреса и ограничивать доступ по региону;
- MAX может отдавать прямые или tracking-ссылки в зависимости от браузера, версии и серверного
  варианта; тесты принимают только заранее подтверждённые адреса и обязательный набор каналов;
- на страницах нет специальных `data-testid`, поэтому используются семантические элементы,
  доступные пользователю тексты, `aria-label`, `placeholder` и стабильные ссылки;
- автотесты не нажимают кнопку отправки СМС и не скачивают установочные файлы;
- параллельный запуск выключен по умолчанию и включается параметрами `parallel` и `threads`.

## Критерии успешного прогона

- все тесты выбранного набора завершились успешно;
- в Allure созданы шаги, данные окружения и вложения;
- тесты не оставили авторизованные сессии и не отправили СМС;
- каждый сценарий может запускаться отдельно и не зависит от порядка выполнения.
